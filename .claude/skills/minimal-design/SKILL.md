# Minimal Design — sub-keeper

무채색·미니멀을 유지하되 **세련되게**. 색이 없을수록 완성도는 타이포 위계·여백·하네어라인·상태 처리에서 나온다. 장식이 아니라 정밀함으로 승부한다.

## 절대 규칙
- **컬러(hue) 사용 금지.** 팔레트는 아래 무채색 램프 하나뿐. 그라데이션·강조색·브랜드 컬러 넣지 않는다.
- 위계는 **색이 아니라 크기·굵기·여백**으로만 만든다.
- 순백/순흑 금지. **오프화이트 배경 + 니어블랙 텍스트**를 쓴다(더 부드럽고 고급스럽다).
- 그림자보다 **하네어라인 보더 + 여백**을 우선한다. 그림자는 쓰더라도 거의 안 보일 정도로만.
- 하나의 시그니처만 강하게: 로고타입/헤딩의 **굵기 대비**. 나머지는 전부 조용하게.

## 디자인 토큰 (CSS 변수 기준값 — 필요 시 미세 조정)
```css
:root {
  /* 무채색 램프 */
  --ink-900: #17181a;   /* 기본 텍스트(니어블랙) */
  --ink-700: #3f4145;   /* 강조 텍스트/아이콘 */
  --ink-500: #6b6e73;   /* 보조 텍스트 */
  --ink-400: #9a9da3;   /* 플레이스홀더/뮤트 */
  --line:    #e6e7e9;   /* 하네어라인 보더 */
  --surface: #ffffff;   /* 카드/입력 배경 */
  --bg:      #fafafa;   /* 페이지 배경(오프화이트) */
  --bg-subtle:#f2f3f4;  /* hover/미묘한 블록 */

  /* 간격 스케일 (4px 베이스) */
  --sp-1: 4px; --sp-2: 8px; --sp-3: 12px; --sp-4: 16px;
  --sp-6: 24px; --sp-8: 32px; --sp-12: 48px; --sp-16: 64px;

  /* 라운드 */
  --radius-sm: 8px;
  --radius:    10px;

  /* 모션 */
  --ease: 160ms cubic-bezier(.2,.6,.2,1);
}
```
> 모바일(Expo)로 확장할 때를 대비해, 이 값들은 한 곳(공유 토큰)에서 관리하고 웹·모바일이 같은 값을 참조한다.

## 타이포그래피
- 굵기는 **최대 3단계**만: 400(본문) / 500(강조·라벨) / 600(헤딩). 700 이상은 로고타입 정도만.
- 스케일: display 32px·600 / h1 24px·600 / h2 20px·600 / body 16px·400(줄높이 1.5) / small 14px / caption 12px·500.
- **문장형 대문자(sentence case)**. 전체 대문자는 caption/eyebrow에서 자간 넓힐 때만 절제해서.
- 본문 폭은 읽기 편하게 제한(약 60~75자). 숫자(금액·총액)는 tabular-nums로 정렬감 확보.

## 레이아웃 & 여백
- 여백이 주 도구다. 인색하게 쓰지 말 것. 섹션·카드 패딩은 --sp-6~--sp-8을 기본으로.
- 간격은 반드시 스케일 값에서만 고른다(임의값 금지).
- 카드/입력: `background: --surface; border: 1px solid --line; border-radius: --radius`.
- 구분은 하네어라인(--line) 또는 여백으로. 굵은 구분선·박스 남발 금지.

## 컴포넌트
- **Primary 버튼**: `bg --ink-900 / text #fff`, hover `--ink-700`, radius --radius, 충분한 패딩(--sp-3 --sp-4).
- **Secondary 버튼**: `bg --surface / border 1px --line / text --ink-900`, hover `bg --bg-subtle`.
- **Text/Ghost 버튼**: `text --ink-700`, hover 밑줄 또는 `bg --bg-subtle`.
- **입력창**: `border 1px --line`, 포커스 시 `border --ink-900` 또는 은은한 링. 플레이스홀더 --ink-400.
- 상호작용 상태(hover/active/focus)는 **회색 단계 이동**으로만 표현(색 변화 금지).

## 상태 처리 (모든 화면 필수)
- **로딩**: 스피너보다 조용한 스켈레톤/플레이스홀더 선호. 레이아웃 밀림 없게.
- **빈 상태**: 사과·감성 문구가 아니라 **다음 행동을 안내**. 예: "아직 등록한 구독이 없어요" + "구독 추가" 버튼.
- **에러**: 사과하지 말고 **무엇이 잘못됐고 어떻게 고치는지**를 담담하게. 인터페이스의 목소리로.

## 문구(카피)
- 능동태·문장형. 버튼은 결과를 말한다: "저장"(O) / "제출"(X). 흐름 내내 같은 단어 유지("삭제" → 토스트 "삭제됨").
- 시스템 용어가 아니라 사용자가 아는 말로. 짧고 구체적으로, 영리함보다 명확함.

## 품질 하한선
- 모바일까지 반응형, 키보드 포커스 가시성 확보, `prefers-reduced-motion` 존중.
- 모션은 --ease로 짧게(150~200ms). 과한 애니메이션은 오히려 완성도를 떨어뜨린다.

## 하지 말 것 (체크리스트)
- [ ] 강조색/그라데이션 추가 — 금지
- [ ] 순흑(#000)/순백(#fff)을 텍스트·배경 대비로 그대로 사용 — 금지(램프 사용)
- [ ] 임의 간격값(예: 13px, 27px) — 금지(스케일 사용)
- [ ] 굵은 그림자·과한 라운드로 "튀게" 만들기 — 금지
- [ ] 위계를 색으로 표현 — 금지(크기·굵기·여백으로)
