# 프론트엔드 PRD — `landing` (핵심 사용자 화면) · 웹 + 모바일

> **모듈 구성**: 프론트엔드를 `landing` / `payment` / `admin` 세 화면군으로 나눈 것 중 **`landing`** 파트.
> `landing` = 로그인 후 사용자가 실제로 쓰는 **핵심 화면군**(대시보드·구독관리·분석·갱신).
> **웹(React) + 모바일(Expo 앱) 둘 다 포함** — 단순 반응형 웹이 아니라, 두 플랫폼의 프론트엔드를 함께 정의한다.
> 결제 흐름 → `payment.md`, 관리자 → `admin.md`, 아키텍처/백엔드/스키마 → `prd-app.md`

---

## 1. 범위

**포함** (이 문서):
- 인증 / 로그인 화면 (소셜 + 이메일)
- 대시보드
- 구독 추가 / 수정 / 삭제
- 카테고리별 분석 차트 *(프리미엄)*
- 갱신 임박 목록
- 내 정보 / 플랜 상태 표시 및 게이팅 UI

**제외** (타 모듈):
- 결제 / 업그레이드 흐름 → `payment.md`
- 관리자 페이지 → `admin.md`
- BaaS 스키마·아키텍처 → `prd-app.md`

---

## 2. 플랫폼 & 공유 원칙

| 레이어 | 선택 | 비고 |
|--------|------|------|
| 웹 | React + Vite | 데스크탑·태블릿 대응 |
| 모바일 | **Expo (React Native)** | iOS·Android, 폰(phone) 우선 |
| 공유 | `packages/core` | 타입·비즈니스 로직·BaaS 클라이언트 (양쪽이 import) |

- **UI는 플랫폼별로 따로**(웹=DOM/CSS, 모바일=RN 컴포넌트), **로직·타입·데이터 접근은 공유**.
- 두 플랫폼 모두 같은 BaaS, 같은 **디자인 토큰**을 사용해 톤 일관성 유지.
- 모바일은 데스크탑 축소가 아니라 **폰 화면에 맞춘 별도 레이아웃**.

---

## 3. 내비게이션

| 플랫폼 | 방식 |
|--------|------|
| 웹 | 좌측 **사이드바** (대시보드 / 구독 / 분석 / 갱신 / 내 정보) |
| 모바일 | 하단 **탭바** (대시보드 / 구독 / 분석 / 갱신 / 내 정보) |

- 모바일 탭바는 엄지 도달 범위(하단) 기준. 주요 액션(구독 추가)은 FAB 또는 탭바 중앙 버튼.

---

## 4. 화면 정의 (웹 vs 모바일)

데이터/로직은 동일, **배치만 플랫폼에 맞게 분리**한다.

| 화면 | 웹 (넓은 화면) | 모바일 (폰, 세로) |
|------|----------------|-------------------|
| 로그인/회원가입 | 중앙 카드 폼 + 소셜 버튼 | 전체 화면 폼, 소셜 버튼 상단 |
| 대시보드 | 총액 카드 + 목록 테이블(다열) + 정렬 선택 | 총액 요약 상단 + 카드 세로 스택 + 정렬 선택 |
| 구독 추가/수정 | 모달 또는 우측 패널 | 전체 화면 폼 |
| 구독 삭제 | 인라인 확인 모달 | 스와이프 액션 또는 확인 시트 |
| 분석 차트 | 넓은 차트 + 범례 옆 배치 | 차트 위 / 범례 아래 세로 스택 |
| 갱신 임박 | 목록 (결제일 임박 순, 필터 없음) | 목록 (결제일 임박 순, 필터 없음) |
| 내 정보/플랜 | 설정 페이지(좌측 항목) | 리스트형 설정 화면 |

### 4.1 로그인 / 회원가입
- 소셜 로그인 + 아이디/비밀번호 둘 다.
- 모바일 소셜 로그인은 리다이렉트/딥링크 설정 필요(BaaS SDK가 대부분 처리).

### 4.2 대시보드
- 등록된 구독 목록, 월/연 환산 총액(통화 혼합 시 결제일 기준 환산 — 로직은 `packages/core`).
- 무료 플랜: 등록 개수 및 10개 제한 안내.
- **정렬 선택**: 낮은 가격순 / 높은 가격순 / 갱신 임박순.
  - 가격순은 통화가 섞인 경우 `packages/core`의 통화 환산 로직으로 기준 통화(**KRW**)로 환산한 값 기준.
  - 정렬 계산 로직은 `packages/core`에 두고 웹·모바일이 공유한다(UI만 각자).

### 4.3 구독 추가/수정/삭제
- 입력 항목: 서비스명, 금액, 통화, 결제 주기, 다음 결제일, 카테고리(드롭다운), 결제수단, 메모.
- 다음 결제일은 결제 주기 기반 자동 계산.

**프리셋 우선(preset-first) 추가 흐름**
- “구독 추가” 진입 시 곧바로 빈 폼을 열지 않고, **검색 가능한 서비스 프리셋 목록**을 먼저 보여준다.
  - 프리셋 데이터는 공유 참조 테이블 `service_presets`(prd-app.md 3.2, 읽기 전용). 조회는 `packages/core` 의 BaaS 클라이언트 경유(`listServicePresets`).
  - 검색은 서비스명 부분 일치(대소문자 무시). 각 항목은 이름 + 카테고리 + 대표 요금/주기(예시값)를 보여준다.
- **프리셋 선택** → 폼에 **서비스명·금액·통화·카테고리·결제 주기**를 자동으로 채운다.
  - 사용자는 **다음 결제일·결제수단**을 입력하고, 자동 채워진 **금액 등도 자유롭게 수정** 가능(요금은 수시로 바뀌므로).
  - 라벨↔CategoryId 매핑, 프리셋→폼 값 변환 로직은 `packages/core` 에 둔다(`categoryIdFromLabel`·`presetToPrefill`, 웹·모바일 공유).
- **목록에 없으면 “직접 입력”** → 프리셋 없이 빈 폼(기존 CRUD)으로 진행.
- 상태: 프리셋 로딩(스켈레톤) / 검색 결과 없음 / 조회 실패(→ “직접 입력” 안내). 무채색·미니멀 유지.
- 플랫폼: 흐름·로직은 공유, UI만 각자(웹=모달, 모바일=전체 화면). **현재는 웹(apps/web)만 구현**, 모바일은 이후.

### 4.4 카테고리 분석 차트 *(프리미엄)*
- 카테고리별 지출 비중(도넛/막대). 집계는 공유 로직, 렌더링은 웹=recharts / 모바일=victory-native 등.
- 무료 사용자에게는 잠금(업그레이드 유도는 `payment.md`로 연결).

### 4.5 갱신 임박 목록
- 기간 필터(7일/30일/90일 등) **없이**, **단일 목록**으로 보여준다.
- 다음 결제일이 임박한 순(**다음 결제일 오름차순**)으로 정렬한다.
- 정렬 계산 로직은 `packages/core`에 두고 웹·모바일이 공유한다(UI만 각자).

### 4.6 내 정보 / 플랜 게이팅
- 현재 플랜(무료/프리미엄) 표시. 프리미엄 전용 기능은 잠금 UI + 업그레이드 진입점(payment로 연결).

---

## 5. 상태 & 데이터

- 모든 데이터 접근은 `packages/core`의 BaaS 클라이언트 경유(웹·모바일 공통).
- 통화 환산·다음 결제일·카테고리 집계 등 계산 로직은 공유.
- **로딩 / 빈 상태 / 에러 상태** UI를 각 화면에 정의(첫 구독 없음, 네트워크 오류 등).
- 접근 제어: 각 사용자는 본인 데이터만.

---

## 6. 플랫폼별 구현 노트

- **웹**: react-router 등 라우팅, 차트 recharts, 반응형(데스크탑/태블릿).
- **모바일(Expo)**: expo-router 라우팅, `expo-notifications`(푸시), 차트 victory-native, 폰 우선 레이아웃.
- **푸시 알림**: 모바일 우선(4.x 화면 자체는 알림 설정 토글 정도만; 발송 로직은 백엔드/`prd-app.md`).

---

## 7. 디자인 원칙

- **정말 심플하고 미니멀**, **강조색 없음**(무채색 기반).
- 여백·타이포그래피 중심 위계.
- 웹/모바일 공통 디자인 토큰(색·간격·타이포) 정의.
- 다크모드는 선택 사항.

## 8. 오픈 이슈 / 확인 필요

1. **제품명** 확정

## 9. 완료 기준 (Acceptance Criteria)

- [ ] 웹(React+Vite)과 모바일(Expo) **둘 다** 핵심 화면 구현
- [ ] 로직·타입·데이터 접근은 `packages/core` 공유, UI만 플랫폼별
- [ ] 웹=사이드바 / 모바일=하단 탭바 내비게이션
- [ ] 로그인(소셜+이메일), 대시보드, 구독 CRUD, 분석 차트, 갱신 임박 동작
- [ ] 무료 10개 제한·프리미엄 게이팅 UI 반영
- [ ] 무채색 미니멀 디자인(강조색 없음), 모바일은 폰 전용 레이아웃(축소 아님)
- [ ] 각 화면에 로딩/빈 상태/에러 상태 처리
