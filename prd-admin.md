# 구독모아 — 관리자 페이지 PRD

- **문서 버전:** v0.1
- **대상 영역:** 운영자용 관리자 대시보드 (회원·구독 상태·매출 지표 관리)
- **상태:** 기획 초안

---

## 0. 공통 컨텍스트 (세 PRD 공유)

- **제품 한 줄:** 흩어진 구독을 한곳에 모아 결제일·총액을 보여주고 새는 돈을 막아주는 구독 관리 도구.
- **요금제:** 무료(구독 10개, 결제일·총액) / 프리미엄(무제한 + 갱신 알림 + 카테고리 분석 + 연간 리포트 + 해지 추천), 월 2,900원 / 연 29,000원, 체험 없음.
- **디자인 방향:** 흰 배경, 무채색 중심, 강조색 없음, 심플·미니멀.
- **반응형 원칙:** 관리자는 **데스크탑 우선**(표·차트·대시보드 중심).
- **기술:** BaaS 기반(아래 권장 참고). 데이터는 회원·구독·결제·플랜 엔티티로 구성.

> **가정 (사용자 확인 필요):** BaaS는 관계형·집계에 유리한 **Supabase(Postgres)** 권장(대안 Firebase). 관리자 권한은 역할 기반 접근(admin role)으로 분리. UI 한글 기본.

---

## 1. 목표

- 운영자가 **사업 건강도(MRR·해지율·프리미엄 이용률)** 와 **회원·구독 상태**를 한 화면에서 파악한다.
- (포트폴리오 관점) 실제 데이터 모델·집계 쿼리·상태 표현을 다룰 줄 안다는 것을 보여준다.

## 2. 접근 제어

- 관리자 전용. 일반 사용자 계정과 분리된 **admin 역할**로만 진입 가능.
- 모든 회원 데이터 조회는 관리자 권한 검증 후에만. (BaaS의 행 수준 보안 정책 또는 서버 함수로 강제)

---

## 3. 화면 구성

### 3.1 지표 대시보드 (메인)

핵심 지표 카드 + 추이 차트.

- **MRR(월 반복 매출)**: 활성 프리미엄 회원의 월 환산 합계. (연 결제는 ÷12로 월 환산)
- **총 회원 / 신규 가입 추이**: 기간별 가입자 수.
- **프리미엄 이용률**: 프리미엄 회원 ÷ 전체 회원.
- **해지율(Churn)**: 기간 내 해지 회원 ÷ 기간 시작 시점 프리미엄 회원.
- **매출 추이**: 월별 결제액 추이 차트.

> 지표 정의는 문서에 고정해 계산 기준을 명확히 한다(특히 MRR 월 환산, 해지율 분모 기준).

### 3.2 회원 목록

- 검색·필터(플랜, 구독 상태, 가입일 범위).
- 컬럼: 닉네임/이메일, **플랜(무료/프리미엄)**, **구독 상태(활성·해지 예정·해지·연체)**, 가입일, 마지막 결제일, 등록 구독 수.
- 행 클릭 → 회원 상세.

### 3.3 회원 상세

- 기본 정보(가입 방식: 소셜/이메일), 현재 플랜·상태, 결제 이력, 등록한 구독 개수(목록 자체는 개인정보라 노출 범위는 미해결 참고).
- 기본 운영 액션(최소): 메모 남기기. (환불·강제 플랜 변경 등은 v1 범위 미해결 참고)

## 4. 데이터 (관리자 관련)

집계는 아래 엔티티에서 파생한다.

```
User       { id, email, auth_provider, plan(free|premium), created_at }
Billing    { user_id, plan_type(monthly|yearly), status(active|canceled|past_due),
             started_at, current_period_end, canceled_at }
Payment    { id, user_id, amount, status, paid_at, provider }
```

- 지표는 원본 테이블에 대한 **집계 뷰/쿼리**로 계산(BaaS view 또는 서버 함수). 대시보드는 읽기 전용.

## 5. 디자인 디테일

- 데스크탑 우선의 표·카드·차트 레이아웃. 무채색이라 차트도 회색 계조로 구분(패턴·굵기·라벨로 가독성 확보).
- 정보 밀도는 높이되 여백으로 정돈. 상태는 색이 아니라 **라벨/뱃지(텍스트)** 로 구분(강조색 없음 원칙 유지).

## 6. 범위 밖 (v1)

- 복잡한 CRM(세그먼트·캠페인), 자동 환불 처리, 권한 세분화(다중 운영자 역할), 데이터 내보내기.

## 7. 미해결

- 회원 상세에서 **개인 구독 목록을 운영자가 볼 수 있게 할지**(개인정보 민감) — 기본은 개수·요약만 노출 권장.
- 환불·강제 다운그레이드 같은 운영 액션을 v1에 포함할지.
- 지표 기간 단위(일/주/월) 기본값.
