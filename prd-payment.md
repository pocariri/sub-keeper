# 구독모아 — 결제 서비스 PRD

- **문서 버전:** v0.1
- **대상 영역:** 프리미엄 정기결제(구독) 판매 및 플랜 생명주기 관리
- **상태:** 기획 초안

---

## 0. 공통 컨텍스트 (세 PRD 공유)

- **제품 한 줄:** 흩어진 구독을 한곳에 모아 결제일·총액을 보여주고 새는 돈을 막아주는 구독 관리 도구.
- **요금제:**
  - 무료: 구독 10개까지, 결제일·총액 보기
  - 프리미엄: 무제한 + 갱신 알림(푸시) + 카테고리 분석 + 연간 리포트 + 해지 추천
  - 가격: **월 2,900원 / 연 29,000원**, 무료 체험 없음
- **디자인 방향:** 흰 배경, 무채색 중심, 강조색 없음, 심플·미니멀.
- **기술:** BaaS + 외부 결제사(아래 권장 참고).

> **가정 (사용자 확인 필요):** 결제사는 KRW 정기결제(빌링키)에 적합한 **토스페이먼츠** 권장(대안: 포트원/아임포트, 해외 대상이면 Stripe). BaaS는 Supabase 권장. 비밀키가 필요한 결제 처리는 **서버(BaaS 서버 함수)** 에서만 수행. 포트폴리오에서는 결제사 **테스트 모드**로 구현.

---

## 1. 목표

- 사용자가 프리미엄으로 매끄럽게 업그레이드하고, **정기결제(시작·갱신·해지)** 가 안정적으로 돌아가게 한다.
- (포트폴리오 관점) 결제사 연동 + **구독 상태 머신** + 기능 게이팅을 제대로 다뤘다는 것을 보여준다.

---

## 2. 무료 / 프리미엄 기능 게이팅

| 기능 | 무료 | 프리미엄 |
|---|---|---|
| 구독 등록 | 10개까지 | 무제한 |
| 결제일·총액 보기 | O | O |
| 갱신 알림(푸시) | X | O |
| 카테고리 분석 차트 | X | O |
| 연간 리포트 | X | O |
| 해지 추천 | X | O |

- 게이팅은 **서버에서 plan 값을 신뢰 소스로** 판정한다(프런트 표시만으로 잠그지 않음).

## 3. 결제 플로우

1. 사용자가 업그레이드 → 플랜 선택(월/연).
2. 결제사 위젯으로 **결제수단 등록 → 빌링키 발급**(카드 원문은 저장하지 않음, PCI는 결제사에 위임).
3. 서버 함수가 첫 결제 승인 → 성공 시 `plan=premium`, `Billing.status=active`, `current_period_end` 설정.
4. 프리미엄 기능 잠금 해제.
5. 주기 도래 시 **자동 갱신**(서버에서 빌링키로 재결제) → `current_period_end` 연장, 결제 이력 적재.

## 4. 구독 상태 머신

```
free
 └─(결제 성공)→ active(premium)
                 ├─(해지 요청)→ canceled → (기간 만료)→ free
                 └─(갱신 실패)→ past_due → (재시도 성공)→ active
                                          └─(재시도 만료)→ free
```

- **해지 정책:** 해지 요청 시 즉시 차단하지 않고 **현재 결제 기간 말까지 프리미엄 유지** 후 무료로 전환(`canceled` → 기간 만료 → `free`).
- **갱신 실패:** `past_due` 진입 후 정해진 횟수/기간 재시도. 회복 실패 시 무료로 다운그레이드.
- **다운그레이드 시 데이터 처리:** 기존 등록 구독은 **모두 유지·조회 가능**. 단 등록 수가 10개를 초과한 상태에서는 **새 구독 추가 불가**(초과분 삭제를 강요하지 않음).

## 5. 결제사·BaaS 연동

- **빌링키 보관·재결제·승인 검증**은 서버 함수에서만. 결제사 시크릿 키는 클라이언트에 노출 금지.
- **웹훅 처리:** 결제 성공/실패/갱신 이벤트를 수신해 `Billing.status`, `current_period_end`, 결제 이력을 갱신. 웹훅 서명 검증 필수.
- **멱등성:** 동일 이벤트 중복 수신에 대비해 결제/갱신 처리는 멱등하게.

## 6. 데이터 (결제 관련)

```
Billing  { user_id, plan_type(monthly|yearly), status(active|canceled|past_due),
           billing_key(서버 전용), started_at, current_period_end, canceled_at }
Payment  { id, user_id, amount, currency(KRW), status(paid|failed|refunded),
           paid_at, provider, receipt_url }
```

- 프리미엄 결제 통화는 **KRW 고정**(사용자가 등록하는 개별 구독의 다중 통화 환산과는 별개 영역).

## 7. 예외·엣지 케이스

- 카드 만료·한도 초과로 갱신 실패 → `past_due` + 사용자 안내(결제수단 갱신 유도).
- 결제 도중 이탈/중복 클릭 → 멱등 처리로 중복 결제 방지.
- 환불 → v1에서는 운영자 수동 처리 가정(자동 환불 UI는 범위 밖).
- 연 결제 중 해지 → 정책상 기간 말까지 유지(부분 환불 없음, v1 기준).

## 8. 보안

- 카드 원문 미저장(결제사 빌링키 사용). 시크릿 키는 서버 환경변수.
- plan/구독 상태는 서버가 신뢰 소스. 프런트는 표시 용도.

## 9. 범위 밖 (v1)

- 부분 환불·일할 정산, 쿠폰·프로모션, 다중 통화 결제, 인앱결제(앱스토어) 연동.

## 10. 미해결

- 갱신 실패 재시도 횟수·간격, `past_due` 유지 기간 기본값.
- 해지 시 즉시 해지 옵션도 줄지(현재는 기간 말 유지만).
- 결제사 최종 확정(토스페이먼츠 vs 포트원).
